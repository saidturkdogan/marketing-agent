AI Content Factory

Basic HTML view of the current architecture: a plan-driven multi-agent pipeline that creates campaign assets, reviews them, and saves results to file + PostgreSQL.

Main Flow
Input Layer CLI (`main.py`) or API (`api.py`) receives topic, platforms, and output types.
→
Planner Builds campaign plan and execution queue with minimum token usage.
`agents/planner.py`
→
Supervisor Routes steps deterministically from the queue instead of relying on LLM routing.
`agents/supervisor.py`
Execution Stages
Researcher Searches and compresses market signals, trends, keywords, and competitor insights.
Strategist Turns research into campaign angle, audience, hooks, CTA, and repurposing plan.
Blog Writer Creates markdown blog content using research + strategy + SEO cues.
Social Writers Instagram, LinkedIn, TikTok, and Twitter/X content packages.
Video Script Writer Builds long-form script and repurposing direction for shorts.
Image Prompt Writer Creates thumbnail, blog visual, and social image prompts.
Reviewer Checks consistency, quality, and publish readiness across all assets.
State and Persistence
Shared State `AgentState` carries `plan`, `assets`, `execution_queue`, `completed_steps`, `target_platforms`, and `requested_outputs`.
File Output Every completed campaign is saved as JSON inside the `outputs/` folder.
PostgreSQL Campaign metadata goes to `campaigns`, generated content goes to `assets`. If `DATABASE_URL` is missing, DB persistence is skipped safely.
